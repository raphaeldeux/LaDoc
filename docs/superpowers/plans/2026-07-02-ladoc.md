# LaDoc Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Mettre en ligne docs.fresquesystemique.org : site Hugo (thème Hextra) documentant le SI Fresque Systémique (LeHub, LeSite, LeBoard), squelette complet puis contenu rédigé par jalons.

**Architecture:** Site statique Hugo buildé sur le VPS depuis `~/LaDoc`, publié dans `/var/www/ladoc`, servi par le Nginx hôte avec HTTPS certbot. Thème Hextra en git submodule (pas de Go requis). Dépôt GitHub `raphaeldeux/LaDoc`.

**Tech Stack:** Hugo extended (dernière release GitHub), thème Hextra (imfing/hextra, submodule), Nginx hôte, certbot, rsync.

**Contrainte transverse :** la doc est publique. Aucun secret, token, mot de passe, IP, port interne ni chemin de credentials dans `content/`. Le jeton GitHub ne doit jamais apparaître dans un fichier commité (il est déjà dans les remotes locaux, c'est tout).

### Task 4 : Déploiement local (deploy.sh + /var/www/ladoc)

**Files:**
- Create: `~/LaDoc/deploy.sh`

- [ ] **Step 1 : Préparer le répertoire cible**

```bash
sudo mkdir -p /var/www/ladoc && sudo chown ubuntu:ubuntu /var/www/ladoc
```

- [ ] **Step 2 : Écrire `deploy.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")"
hugo --minify --gc --printPathWarnings
rsync -a --delete public/ /var/www/ladoc/
echo "✅ Déployé → https://docs.fresquesystemique.org"
```

- [ ] **Step 3 : Rendre exécutable et lancer**

Run: `chmod +x deploy.sh && ./deploy.sh && ls /var/www/ladoc/index.html`
Expected: build + rsync OK, index.html présent

```bash
git add deploy.sh && git commit -m "Script de déploiement"
```

### Task 5 : Nginx + HTTPS

**Files:**
- Create: `/etc/nginx/sites-available/docs-fresquesystemique`

- [ ] **Step 1 : Écrire la conf Nginx (calquée sur les autres sites statiques du VPS — regarder `boussolecommune` ou `maintenant-fresquesystemique` comme référence de style)**

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name docs.fresquesystemique.org;

    root /var/www/ladoc;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
```

- [ ] **Step 2 : Activer et recharger**

```bash
sudo ln -s /etc/nginx/sites-available/docs-fresquesystemique /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

- [ ] **Step 3 : HTTPS**

```bash
sudo certbot --nginx -d docs.fresquesystemique.org --non-interactive --agree-tos -m claude@raphael-deux.fr --redirect
```
